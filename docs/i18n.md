# docs/i18n.md — Kohelet

> Documento de internacionalización.  
> Define cómo deben manejarse textos visibles, claves, idioma inicial y reglas para evitar hardcodear UI.

**Proyecto:** Kohelet  
**Editor:** Sofer  
**Estado:** inicial  
**Última revisión:** 2026-05-24

---

## 1. Objetivo

Kohelet inicia en español argentino:

```text
es-AR
```

Pero la arquitectura debe permitir sumar otros idiomas después de 1.0.

Regla central:

```text
Todo texto visible de UI debe salir de src/i18n/locales/es-AR.json.
```

---

## 2. Excepciones permitidas

Pueden quedar como literales:

```text
Kohelet
Sofer
```

También pueden quedar como literales técnicos no visibles para usuario final:

- nombres internos de tipos,
- nombres de archivos,
- claves de configuración,
- logs de desarrollo.

---

## 3. Ubicación

```text
src/i18n/
  i18n.ts
  locales/
    es-AR.json
```

Reglas:

- Las claves deben ser estables.
- Agrupar por módulo.
- No crear textos duplicados si una clave común alcanza.
- No construir frases complejas concatenando pedazos si puede evitarse.

## 12. Accesibilidad

Las etiquetas de accesibilidad también deben estar en i18n cuando sean visibles/semánticas.

```json
{
  "a11y": {
    "openContextPanel": "Abrir panel contextual",
    "closeContextPanel": "Cerrar panel contextual",
    "openNavigationTree": "Abrir árbol narrativo",
    "closeNavigationTree": "Cerrar árbol narrativo"
  }
}
```

---

## 13. Interpolación

Usar interpolación para valores dinámicos.

Ejemplo:

```json
{
  "goals": {
    "wordCount": "{{count}} palabras"
  }
}
```

Evitar:

```tsx
{count + ' palabras'}
```

---

## 14. Reglas para componentes

Componentes React:

- no deben hardcodear textos visibles,
- deben usar función/helper de traducción,
- deben tener labels accesibles traducidos,
- deben evitar concatenar frases traducibles,
- deben mantener claves estables.

Permitido:

```tsx
<button aria-label={t('editor.toolbar.bold')}>
  {t('editor.toolbar.bold')}
</button>
```

Evitar:

```tsx
<button aria-label="Negrita">Negrita</button>
```

---

## 15. Testing i18n

Tests recomendados:

- detectar claves faltantes críticas,
- verificar labels de toolbar,
- verificar labels de tipos de escena,
- verificar mensajes de storage/export,
- smoke test de render sin textos undefined.

---

## 16. Checklist para Codex

Antes de cerrar una tarea UI:

- [ ] Se usó la skill `i18n-ui` si hay textos visibles.
- [ ] No quedaron textos visibles hardcodeados.
- [ ] Se agregaron claves a `es-AR.json`.
- [ ] Se usaron claves agrupadas por módulo.
- [ ] Los `aria-label` relevantes están traducidos.
- [ ] No se concatenaron frases traducibles.
- [ ] Se actualizaron tests si corresponde.
